# E-Portfolio Landing Page - Setup Guide

Your professional E-Portfolio landing page has been created with 3 sections:

## 📋 Sections Included:

1. **Hero Section** - Introduction with your profile photo, name, and headline
2. **Projects Section** - Showcase your portfolio projects with cards
3. **Call-to-Action Section** - Encourage visitors to connect with you

## 🎨 Files Created:

- `index.html` - Main landing page structure
- `styles.css` - Complete styling and responsive design

## ⚙️ Customization Steps:

### 1. **Add Your Profile Photo**
   - Replace `profile.jpg` in the HTML with your actual photo
   - Place your image in the same folder as `index.html`
   - Or update the `src` path in the HTML

### 2. **Update Project Cards**
   - Replace "PROJECT TITLE" with your actual project names
   - Update "Project Brief Description" with project summaries
   - You can add more project cards by duplicating the `project-card` div

### 3. **Update Contact Links**
   - Update `Resume` button link to your resume URL/file
   - Update `LinkedIn` button link to your LinkedIn profile
   - Email link `irahlzrd@gmail.com` is already in the CTA section

### 4. **Customize Navigation**
   - Update link destinations in the nav (`#about`, `#projects`, `#contact`)
   - Add smooth scrolling to sections or external pages

### 5. **Personalize Content**
   - Update the hero section text with your bio
   - Change the logo text from "ir" to your initials
   - Update email address to your actual email

## 🚀 Features:

✅ Fully responsive design (mobile, tablet, desktop)
✅ Clean, modern aesthetic matching your mockups
✅ Smooth scrolling navigation
✅ Hover effects on interactive elements
✅ Professional typography and spacing
✅ Blue gradient backgrounds with subtle patterns

## 📱 Responsive Breakpoints:

- Desktop: Full layout
- Tablet (1024px): Adjusted spacing
- Mobile (768px): Stacked layout, optimized touch targets

## 💡 Tips:

- Use high-quality photos (JPG/PNG)
- Keep project descriptions concise (1-2 lines)
- Test on mobile devices before going live
- Consider adding actual project mockups to replace placeholder boxes
